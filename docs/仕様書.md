# 漢字学習アプリ 仕様書

子供（小学1〜6年生）が漢字を楽しく覚えるための Web アプリケーション。

> 本ドキュメントは「仕様決め」フェーズの成果物です。実装は含みません。

---

## 1. 目的・コンセプト

- 小学生が **ひらがな + イラスト** をヒントに、対象の漢字を **選択** または **手書き** で答える。
- 辞書機能で **意味・例文** を確認できる。
- **正解回数・学習状況** を記録して達成感を得られる。
- **ひらがなの名前 + 暗証番号（PIN）** による簡易認証で、子供ごとに実績を保存する。

### 想定ユーザー
- 主対象: 小学1〜6年生の子供（ひらがなが読める）。
- 副対象: 保護者（進捗の確認）。

### 設計方針
- 子供が一人で操作できる **やさしい UI**（大きなボタン、ひらがな表記、少ない文字）。
- スマホ・タブレット優先（手書きのしやすさ）。PC も対応。
- サーバーレス構成で **低コスト・低運用負荷**。

---

## 2. 機能要件

### 2.1 認証
- ログイン画面で **ひらがなの なまえ** と **4桁の あんしょうばんご（PIN）** を入力。
- 新規なら「はじめてのひと」ボタンで登録（なまえ + PIN + 学年）。
- 認証はあくまで「実績をひもづける」ための簡易的なもの（本格的なセキュリティは目的外）。
- ログイン状態はブラウザに保持（トークン）。

### 2.2 出題モード
学年（1〜6年）と出題範囲を選んでスタート。1問ごとに以下を提示:

- 対象の漢字の **読み（ひらがな）**
- 意味を補助する **イラスト**（例: 「やま」→ 山のイラスト）
- 回答方法は **モード選択**:
  1. **えらぶモード（選択式）**: 4択などの候補から正しい漢字を選ぶ。
  2. **かくモード（手書き）**: キャンバスに指/ペンで漢字を書いて答える。

回答後、正誤を表示し、正解の漢字・読み・意味・例文を見せる。

### 2.3 手書き認識（→ ブラウザ内 自動採点に決定 / KanjiVG方式）
- HTML5 Canvas で描画を取得し、**ストロークを点列として記録**する。
- **採点は端末内（ブラウザ）で自動。追加コストなし・ネット不要。** クラウドOCRは使わない。
- 参照データは **KanjiVG**（全漢字の正しい筆順SVG・CC BY-SA 3.0）。ビルド時に
  `scripts/build-strokes.mjs` で各ストロークを正規化・サンプリングして
  `strokes-grade-N.json` を生成しておく。
- 採点ロジック（`web/src/scoring.js`）は3つの信号を合成:
  1. **形の重なり**（お手本骨格と書いた線をグリッド化して Dice 係数）
  2. **画数**（ストローク数が合っているか）
  3. **筆の向き**（各画の始点→終点方向のコサイン類似）※画数が合うときのみ
- 結果は段階評価: 💯かんぺき / ⭕できた / ✨おしい / 💪もういちど。
  「かんぺき・できた」を正解として記録。しきい値は `scoring.js` 先頭で調整可能。
- 誤判定に備え「かけてた！せいかいにする」等の**手動オーバーライド**ボタンを残す。
- 副次効果として **正しい画数・筆順を促す**（学習効果）。

### 2.4 辞書
- 漢字ごとに: 読み（音・訓）、意味、例文（1〜3個）、学年、画数、部首（任意）。
- 一覧・検索（読み/学年でフィルタ）。
- その漢字の **自分の正解回数 / 挑戦回数** を表示。

### 2.5 実績・進捗
- 漢字ごとの **挑戦回数・正解回数・最終学習日**。
- 学年ごとの **習得率**（例: 3年生 45/200 字）。
- 連続学習日数（ストリーク）などのモチベーション要素（任意）。

## 3. システム構成（AWS）

```
[ブラウザ / SPA]
      │  静的配信
      ▼
[CloudFront] ──→ [S3]  （HTML/JS/CSS・イラスト・漢字データ）
      │  /api/*
      ▼
[API Gateway] ──→ [Lambda] ──→ [DynamoDB]（ユーザー・実績）
```

### 3.1 フロントエンド配信
- **S3**: SPA（静的ファイル）とアセット（イラスト、漢字マスタ JSON）を配置。
- **CloudFront**: S3 をオリジンとして配信。HTTPS 化、キャッシュ、SPA 用に 403/404 を `index.html` へフォールバック。
- S3 バケットは **非公開**、CloudFront の **OAC（Origin Access Control）** 経由でのみアクセス。

### 3.2 バックエンド（動的データ）
- **API Gateway (HTTP API)** + **Lambda** で REST API を提供。
- CloudFront のビヘイビア `/api/*` を API Gateway にルーティング（同一ドメインで CORS 回避）。
- **DynamoDB** にユーザーと実績を保存。

### 3.3 漢字マスタデータの扱い
- 漢字マスタ（読み・意味・例文・学年など）は **静的 JSON として S3 に配置**（更新頻度が低く、動的である必要がないため）。学年ごとにファイル分割。
- 実績など **ユーザーごとに変わるデータのみ DynamoDB**。

## 4. データ設計

### 4.1 DynamoDB テーブル（シングルテーブル設計）

テーブル名: `KanjiApp`

| 用途 | PK | SK | 主な属性 |
|------|----|----|---------|
| ユーザー | `USER#<userId>` | `PROFILE` | name(ひらがな), pinHash, grade, createdAt |
| 実績（漢字別） | `USER#<userId>` | `KANJI#<漢字>` | attempts, corrects, lastStudiedAt |
| ログイン索引 | `LOGIN#<name>` | `PROFILE` | userId（なまえ→ユーザー解決用） |

- `userId`: サーバー採番（UUID）。
- **PIN はハッシュ化して保存**（平文保存しない）。同名ユーザーは name + PIN の組で識別、または登録時に重複名を許容し userId で一意化。
- 集計（学年ごとの習得率）は実績レコードをクエリして算出。

### 4.2 漢字マスタ JSON（S3・例）

```json
{
  "grade": 1,
  "kanji": [
    {
      "char": "山",
      "yomi": ["やま", "サン"],
      "meaning": "つちが たかく もりあがった ところ。",
      "examples": ["やま に のぼる", "ふじさん"],
      "strokes": 3,
      "grade": 1
    }
  ]
}
```

- 学年別: `kanji/grade-1.json` … `kanji/grade-6.json`。
- **初回スコープは 1〜2年生（80 + 160 = 240字）** から着手し、以降の学年は順次追加する。
- 対象字数の目安（学年別配当漢字・合計1026字）:

| 学年 | 字数 |
|------|------|
| 1年 | 80 |
| 2年 | 160 |
| 3年 | 200 |
| 4年 | 202 |
| 5年 | 193 |
| 6年 | 191 |

## 5. API 仕様（案）

| メソッド | パス | 説明 | 認証 |
|---------|------|------|------|
| POST | `/api/signup` | 登録（name, pin, grade） | 不要 |
| POST | `/api/login` | ログイン（name, pin）→ token | 不要 |
| GET | `/api/me` | 自分のプロフィール | 要 |
| GET | `/api/progress` | 実績一覧・習得率 | 要 |
| POST | `/api/answer` | 回答結果を記録（char, correct） | 要 |
| GET | `/api/kanji/:char` | 辞書 + 自分の実績 | 任意 |

- 認証: ログインで発行するトークン（JWT 等）を `Authorization` ヘッダで送信。
- 漢字マスタ自体は S3 の静的 JSON を直接取得（API を介さない）。

## 8. 決定事項 / 未決事項

### 決定済み
1. **手書きの採点方式 = ブラウザ内 自動採点**（KanjiVG の筆順データと照合し、形・画数・筆順で段階評価）。追加コストなし・ネット不要。手動オーバーライドも用意。
2. **イラストは生成 AI で用意**。初回スコープは **1〜2年生（240字）** から。
   - 画風を統一するプロンプト方針を決め、字ごとに生成 → S3 配置。ライセンス/利用規約は確認する。
3. **初回スコープ = 1〜2年生**。3〜6年は順次追加。
4. **PWA（オフライン対応）・保護者ビューは今回は見送り**（将来拡張）。

### 未決（実装時に詰める）
1. **認証の強度** … PIN の総当たり対策（試行回数制限）をどこまでやるか。子供向け前提で最小限。
2. **イラスト生成の具体手段** … 使用する生成 AI・画風統一プロンプト・命名規則（例: `illustrations/山.png`）。
3. **コスト試算** … 個人利用ならほぼ無料枠内の見込み（要確認）。
